Changelog for GemiusSDK iOS:

VERSION 2.2.0
Build date: 21.07.2025

Changes:
- Added new dataProcessingConsentSource property, allowing specification of the consent source used for data
  processing (Manual, CMP, or TC String)
- Added support for manual setting of the IAB-compatible TC String
- Removed the useCMP property introduced in the previous version (replaced with dataProcessingConsentSource)
- Added device model information to User Agent strings
- Added reporting of user-visible app name
- Updated SDK's logging mechanism to use os_log
- Updated Google Mobile Ads dependency to version 12.7.0
- Added new GemiusSdkSample sample app illustrating SDK usage with SwiftUI and SPM

--------------------------------------------------------------------------------

VERSION 2.1.0
Build date: 08.01.2025

Changes:
- Added support for iAB TCF 2.2
- Timestamps are being sent with every hit
- Methods addExtraParameter and setExtraParameters from BaseEvent class check for nil values and log errors when appropriate
- Updated GoogleMobileAds dependency to 11.13.0 version

--------------------------------------------------------------------------------

VERSION 1.3.0 
Build date: 24.01.2018

- official support for iOS 11 (tested up to 11.2) (previous SDK versions should be compatible too, but only this one was fully tested on the new OS)

Fixes:
- fixed memory leak for AdOcean

Changes:
- added original WebView client name (typically "Mozilla/5.0") in front of User-Agent returned by getUA4WebView(). For wider compatibility, for example fixes problem with "login with Facebook".

- added onContentReady() callback, informing when static contents of an ad (HTML5, images, GIFs) have been loaded. Useful for preloading billboard ads in the background. See documentation for details.

- more verbose logging of Audience requests errors.
